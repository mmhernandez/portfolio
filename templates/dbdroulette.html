Dead By Daylight Perks Roulette

Python | Flask | MySQL | Javascript | API | Bootstrap | CSS | Jinja2 | HTML

I was commissioned to create this app for a group of gamers who needed a better way to view Dead By Daylight game perks, randomly generate perks for their preferred gameplay, and save any perk builds to a favorites list for future reference. I was able to do that, and more!

I used a couple of different libraries and packages to achieve various goals for this app, including Cloudinary for image storage and retrieval, WTForms to include a reCAPTCHA check for user signups, the GoogleAPI for an alternate user login option, and Bcrypt for password hashing.

Project Details

The Dead By Daylight Perks Roulette app offers many features, available to any user. This includes full perks lists with filter and search options, perks roulette which selects 4 perks at random for custom game play, and a modified roulette option with 6 randomly selected perks for the user to select their preferred 4 for game play.

Users can sign-up to create a new account, where I implemented a reCAPTCHA check to avoid bots. Or, if preferred, users can login with Google. By importing the Google-oauth library, the app can authenticate users with their gmail credentials for both ease of use and additional security.

There are a few additional app features that are only available to logged in users. After logging in, users can view a Favorites tab in the left navigation pane as well as new Add to Favorites buttons on each of the roulette pages. From this Favorites section, users can view any favorites they've created and create custom builds to add to their favorites list. Because this favorites feature adds entries to a database, I decided to nest it behind a login (and also included logic to limit the number of favorites each user can create).

Speaking of special features - this app was also built with administration in mind. Each user has a user-level associated, which is defaulted to a user level upon account creation. Any super admin user has the ability to change user permissions, and can elevate users to admin or super admin level. Admin and super admin users can create/edit/delete perks, but only super admin users can modify user permission levels.

Application Deployment

This project was deployed to AWS in an Amazon EC2 instance. A domain was purchased from Amazon and connected to the EC2 instance using the Amazon Route53 service. An SSL certificate was also purchased; using the Load Balancer tool, traffic is routed to HTTPS and references the cert for a secure site experience for users.
